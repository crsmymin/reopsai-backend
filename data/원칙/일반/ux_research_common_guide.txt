UX 리서치 방법론 공통 가이드

========================================

[UX 리서치 진행 과정 공통 가이드]

대부분의 UX 리서치 방법론은 다음과 같은 공통적인 진행 과정을 따릅니다:

준비하기 (1~3주)
- 리서치 계획서 작성 (리서치 질문, 테스트 형식)
- 참가자 조건 확정 및 모집
- 디스커션 가이드 작성하기
- 파일럿 테스트
- 사전 준비 (비밀유지계약서 작성, 레코딩 기기 설치)

데이터 수집하기 (1~5일)
- 테스트 목적과 주의사항 안내
- 시나리오와 태스크 전달하고 진행
- 설문조사 (선택사항)
- 관찰자들과 디브리핑

분석하기 (2~4주)
- 데이터 리뷰, 데이터 코딩
- 패턴 찾기, 추론하기
- 리포트 준비 및 발표

========================================

[사용성 평가 요소 공통 가이드]

사용성은 크게 세 가지 요소로 평가합니다:

1. 유효성(Effectiveness)
- 목표 달성의 정확성과 완전성
- 참가자가 태스크를 성공적으로 완료했는가?
- 오류가 얼마나 발생했는가?

2. 효율성(Efficiency)
- 목표 달성에 필요한 자원 대비 성과
- 참가자가 얼마나 빠르고 효율적으로 태스크를 수행했는가?
- 과도한 인지적·정신적 에너지가 필요한가?

3. 만족도(Satisfaction)
- 사용 경험에 대한 주관적 수용성
- 참가자가 프로덕트 사용 과정에 대해 얼마나 만족했는가?

========================================

[리서치 방법론 분류 기준]

1. 질적 리서치 VS 양적 리서치

질적 리서치(Qualitative Research)
- 사용자의 맥락, 동기, 감정, 문제 인식 등 깊이 있는 인사이트를 도출하는 데 초점
- 데이터는 인터뷰 전사, 관찰 메모, 사용자 저널 등 텍스트나 서술적 형태로 수집
- 예시: 심층 인터뷰, 다이어리 스터디, 에스노그라피

양적 리서치(Quantitative Research)
- 특정 현상의 규모, 빈도, 통계적 유의성을 검증하는 데 초점
- 데이터는 수치, 비율, 그래프 형태로 표현되며, 패턴과 일반화 가능성을 제공
- 예시: 설문조사, 총괄적 사용성 테스트, A/B 테스트

2. 태도적 리서치 VS 행동적 리서치

태도적 리서치 (What They Say?)
- 사용자의 생각, 의견, 기대, 동기를 파악하는 방법
- 질문·설문·토론을 통해 주관적 진술을 수집
- 예시: 설문조사, 인터뷰, 포커스 그룹

행동적 리서치 (What They Do?)
- 사용자의 실제 행동, 과업 수행 과정, 시각적 주의 등을 직접 관찰하여 데이터화
- 사용자가 말한 것과 실제 행동이 다를 수 있기 때문에 행동 데이터가 신뢰도 보완 역할을 한다
- 예시: 사용성 테스트, 아이트래킹, 로그 데이터 분석

3. 탐색적 리서치 VS 평가적 리서치

탐색적 리서치 (Exploratory Research)
- 제품/서비스 초기 단계에서 사용자의 니즈, 문제, 맥락을 발견하기 위한 방법
- 아직 명확히 정의되지 않은 기회 영역을 찾거나 아이디어를 발전시키는 데 적합
- 예시: 에스노그라피, 다이어리 스터디, 컨텍스추얼 인쿼리

평가적 리서치 (Evaluative Research)
- 프로토타입, 개발 완료 단계, 출시 이후 등에서 디자인의 사용성·효과성을 검증하는 방법
- 사용자가 실제로 과업을 수행하면서 겪는 문제를 파악해 개선점을 도출
- 예시: 형성적/총괄적 사용성 테스트, 휴리스틱 평가, 아이트래킹

========================================

[리서치 방법론별 특성 요약]

탐색적 리서치
- 심층 인터뷰: 왜'라고 물으며 근원에 접근 (태도적, 질적, 탐색적, 4~6주)
- 설문조사: 사용자 의견과 태도 측정하기 (태도적, 양적, 탐색적&평가적, 4~8주)
- 컨텍스추얼 인쿼리: 사용자 맥락에서 작업 흐름 관찰하기 (태도적&행동적, 질적, 탐색적, 6~8주)
- 다이어리 스터디: 일상에서 사용자 경험 엿보기 (태도적&행동적, 질적, 탐색적&평가적, 6~10주)
- 카드소팅: 사용자의 인지 구조 이해하기 (태도적, 질적&양적, 탐색적&평가적, 2~4주)
- 에스노그라피: 사용자 일상 속에 잠수하기 (태도적&행동적, 질적, 탐색적, 8~12주)
- 사용자 참여 디자인: 사용자와 함께 디자인하기 (태도적, 질적, 탐색적, 2~4주)
- 포커스 그룹 디스커션: 다양한 사용자 의견을 한자리에서 듣기 (태도적, 질적, 탐색적, 4~6주)

평가적 리서치
- 형성적 사용성 테스트: 사용자가 어디서 헤매는지 파악하기 (행동적, 질적, 평가적, 2~4주)
- 총괄적 사용성 테스트: 전체적인 프로덕트 성능 측정하기 (행동적, 양적, 평가적, 4~6주)
- 아이트래킹: 사용자 시선이 머무는 곳 파악하기 (행동적, 질적&양적, 평가적, 4~6주)
- 콘셉트 평가: 아이디어가 니즈에 부합하는지 알아보기 (태도적, 질적, 탐색적&평가적, 2~3주)
- 게릴라 사용성 테스트: 빠르게 사용자 의견 묻기 (태도적&행동적, 질적, 평가적, 1~3일)
- 휴리스틱 평가: 사용성 원칙 관점에서 프로덕트 평가하기 (행동적, 질적, 평가적, 1주)

========================================

[효과적인 리서치를 위한 공통 원칙]

1. 목표 중심 설계
- 명확한 리서치 목표 설정
- 목표에 적합한 방법론 선택
- 측정 가능한 성공 기준 정의

2. 참가자 중심 접근
- 타겟 사용자 그룹 명확히 정의
- 적절한 참가자 수 확보
- 참가자 편의성 고려

3. 체계적 실행
- 일관된 프로토콜 적용
- 데이터 품질 관리
- 윤리적 가이드라인 준수

4. 통합적 분석
- 정량적/정성적 데이터 병행 분석
- 패턴 및 트렌드 식별
- 실행 가능한 인사이트 도출

5. 지속적 개선
- 리서치 결과를 다음 단계에 반영
- 방법론의 효과성 평가
- 조직 내 리서치 역량 강화
