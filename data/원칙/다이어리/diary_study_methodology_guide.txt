다이어리 스터디 방법론 설계 가이드

========================================

[다이어리 스터디 개념과 활용 목적]

맥락적 이해 확보
- 사용자가 어떤 상황에서 특정 행동(예: 서비스 탐색, 가입 고려, 앱 사용 등)을 하는지 그 배경과 계기를 알 수 있음
- 단순한 순간의 행동 관찰이 아니라, 일상 속 반복적/누적적 경험을 통해 서비스 이용 동기를 깊이 이해

행동·감정 곡선 추적
- 시간이 지남에 따라 사용자의 감정 변화와 행동 패턴을 시각적으로 분석할 수 있음
- 예를 들어 보험 가입 고려 단계에서 불안이 언제 높아지고, 어떤 순간에 낮아지는지 확인 가능

사용성 테스트 전 보강 자료
- 실험실 기반 사용성 평가나 인터뷰 전에, 실제 생활 속 경험 맥락을 확보하여 질문이나 과업 설계를 더 정교하게 만들 수 있음

잠재적 니즈와 Pain Point 발견
- 사용자가 직접 기록하기 때문에 표면적으로 드러나지 않는 불편이나 잠재적 기대를 포착할 수 있음
- 단기적 태스크 기반 리서치에서 놓칠 수 있는 장기적 패턴을 파악하는 데 유용

========================================

[다이어리 스터디 방법론의 포괄적 가이드]

다이어리 스터디는 인터뷰나 설문조사와 달리, 참가자에게 많은 시간과 집중을 요구하는 기법

장기 리서치의 특성상 중도 포기 혹은 저조한 참여도와 같은 문제 발생 가능

참가자들의 지속적, 적극적 참여 독려가 필요함

========================================

[다이어리 스터디 참여 독려 방법 가이드]

중도포기 인원을 고려해, 10~15%의 추가 인원 모집

지나친 반복적 기록 요구를 지양해야 함

지속적 알림과 격려를 통해 과제 리마인드 필요

참가자가 과제를 이해하지 못하거나 헤멜 경우를 대비해 질문할 수 있는 연락망 제공

기록이 제대로 작성되지 않았다면 미리 공지한 연락망을 통해 추가 질문

다이어리 스터디를 두 세 차례로 나누어 진행하며, 각 일정에 따라 인센티브 지급 방법

보통 참가자 인센티브는 리서치 마지막에 지급하지만 다이어리 스터디의 경우에는 지속적인 참여를 이끌기 위해 스터디를 두세 차례 나누어 진행하고 각 일정에 따라 인센티브를 지급

========================================

[다이어리 스터디의 강점과 한계]

장점:
- 참가자가 경험한 순간을 자연스러운 환경에서 즉시 기록 가능
- 참가자의 평소 습관과 일과 파악 가능
- 여러 참가자의 관찰 데이터를 동시 수집하기 때문에, 데이터를 각각 취합해야 하는 직접 관찰보다 효율적
- 장시간에 걸쳐 진행되므로 사용자 행동이나 태도의 변화를 추적할 수 있음
- 리서처의 개입 없이 참가자 스스로 기록하기 때문에 호손 효과를 줄일 수 있음

한계:
- 참가자 기록에 의존하므로, 참가자의 성실하게 임하지 않을 경우 데이터의 실효성이 떨어짐
- 참가자의 생각과 태도를 자세히 물을 수 없음. 이를 보완하기 위해 스터디 중간 지점이나 스터디 종료 후에 추가 인터뷰를 진행하기도 함
- 리서치 준비, 데이터 수집, 분석에 상당한 시간이 소요됨

========================================

[다이어리 스터디 진행 과정 가이드]

리서치 진행 과정에 대한 공통 가이드는 ux_research_common_guide.txt를 참조하세요.

다이어리 스터디 특화 사항:
준비하기(2~3주)
- 커뮤니케이션 방식 선택하고 지시사항 준비

다이어리 기록하기(1~4주)
- 다이어리 스터디 시작(스터디 목적과 참가 방법 등 설명)
- 참가자가 성실하게 기록하고 있는지 중간 점검
- 필요시 스터디 중간 지점 혹은 마지막에 일대일 인터뷰 진행

분석하기(2~4주)
- 데이터 리뷰, 데이터 코딩
- 패턴 찾기, 추론하기

========================================

[다이어리 스터디 설계 고려사항]

1. 참가자 모집 전략
- 타겟 사용자 그룹 명확히 정의
- 충분한 모집 인원 확보 (중도 포기율 고려)
- 참가자 동기 부여 방법 설계
- 지속적 참여를 위한 인센티브 체계 구축

2. 기록 방법 선택
- 텍스트 기반 기록
- 사진/영상 기록
- 앱 기반 기록
- 하이브리드 방식

3. 질문 설계 원칙
- 구체적이고 명확한 질문
- 과도한 부담을 주지 않는 분량
- 일상적 맥락에 자연스럽게 녹아들 수 있는 질문
- 시간대별/상황별 맞춤 질문

4. 참가자 관리
- 정기적 체크인 및 격려
- 질문 및 피드백 채널 운영
- 중간 점검 및 데이터 품질 관리
- 참여도 모니터링

========================================

[다이어리 스터디 질문 구성 가이드]

1. 맥락 정보 수집
- 시간, 장소, 상황 기록
- 함께 있던 사람들
- 행동 전후 상황

2. 행동 기록
- 구체적인 행동 내용
- 사용한 도구나 방법
- 소요 시간
- 결과나 반응

3. 감정 및 생각 기록
- 그 순간의 감정
- 생각이나 고민
- 만족도나 불만
- 기대나 우려

4. 환경적 요인
- 물리적 환경
- 사회적 맥락
- 기술적 환경
- 외부 영향 요인

========================================

[다이어리 스터디 데이터 분석 방법]

1. 정량적 분석
- 행동 빈도 및 패턴 분석
- 시간대별 활동 분석
- 감정 변화 추이 분석
- 참가자별 특성 분석

2. 정성적 분석
- 주제별 코딩 및 분류
- 패턴 및 트렌드 식별
- 인사이트 도출
- 스토리텔링 구성

3. 시각화
- 타임라인 구성
- 감정 곡선 도표
- 행동 패턴 맵
- 사용자 여정 지도

========================================

[다이어리 스터디와 다른 방법론과의 결합]

1. 인터뷰와 결합
- 다이어리 기록 후 심화 인터뷰
- 기록 내용에 대한 추가 질문
- 맥락적 이해 보완

2. 사용성 테스트와 결합
- 다이어리 결과를 테스트 시나리오로 활용
- 실제 사용 맥락 반영
- 더 현실적인 태스크 설계

3. 설문조사와 결합
- 다이어리 패턴을 바탕으로 한 설문 설계
- 정량적 검증
- 일반화 가능성 확인

========================================

[다이어리 스터디 성공을 위한 핵심 원칙]

1. 참가자 중심 설계
- 참가자의 일상에 자연스럽게 녹아들 수 있는 설계
- 과도한 부담을 주지 않는 분량과 빈도
- 명확하고 이해하기 쉬운 지시사항

2. 지속적 관리
- 정기적인 체크인과 격려
- 질문 채널 운영
- 데이터 품질 모니터링

3. 유연한 접근
- 참가자별 특성 고려
- 상황에 따른 조정
- 중간 피드백 반영

4. 체계적 분석
- 명확한 분석 프레임워크
- 정량적/정성적 분석 병행
- 인사이트 도출 및 검증

5. 실용적 활용
- 구체적이고 실행 가능한 결과
- 이해관계자와의 공유
- 후속 연구나 개선에의 활용
