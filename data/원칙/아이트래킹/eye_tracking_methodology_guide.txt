아이트래킹(Eye-Tracking) 방법론 가이드

========================================

[아이트래킹 방법론의 정의]

아이트래킹(Eye-Tracking) 방법론은 사용자의 시각적 주의와 인지 과정을 정량적으로 측정하기 위해 활용된다. 사용자가 화면이나 제품 인터페이스를 볼 때 시선이 머무는 위치와 이동 경로를 추적하여, 어떤 요소가 먼저 주목받는지, 어떤 구간에서 이해나 탐색에 어려움을 겪는지 파악할 수 있다.

========================================

[아이트래킹 사용 이유]

객관적 데이터 확보
설문이나 인터뷰와 달리, 사용자가 어디를 얼마나 보는지에 대한 무의식적 반응을 직접 측정할 수 있어 주관적 편향을 줄일 수 있다.

사용성 문제 진단
버튼, 메뉴, 안내 문구 등 핵심 요소가 실제로 사용자의 주목을 받는지 확인함으로써, 정보 구조나 시각적 디자인의 개선 포인트를 도출할 수 있다.

인지 과정 이해
사용자가 과업을 수행할 때 어떤 순서로 정보를 탐색하고 의사결정을 하는지 시각적 전략(visual strategy)을 파악하여, 효율적인 UX 설계 근거를 제공한다.

========================================

[아이트래킹의 강점과 한계]

장점:
- 사용자가 웹페이지에서 집중하는 부분을 알 수 있음
- 사용자가 콘텐츠를 읽는지 아니면 단지 훑어보는지 알 수 있음
- 사용자가 실제로 무엇을 찾고 있는지 파악할 수 있음. 탐색하는 대상이 명확하게 보이지 않으면 사람이 동공 크기가 커지는데 이러한 현상은 사용자가 카테고리 종류가 많은 새로운 웹사이트에 접속했을 때 나타남
- 여러 사용자 그룹의 웹사이트 이용 패턴을 비교할 수 있음

한계:
- 아이트래킹 장비가 필요하고 결과를 분석하는 데 전문적인 지식이 요구됨
- 모바일 화면은 아이트래킹을 하기 어렵기 때문에 웹사이트에 적합함
- 사용자가 특정 부분에 시선을 두더라도 주의 깊게 살펴보는 것인지 확신하기 어려움. 짧은 시간 동안 별 생각없이 시선을 고정하는 경우도 있음
- 사용자가 대상을 바라보는 이유를 알 수 없음
- 참가자 특성이 결과에 영향을 미칠 수 있음. 사용자가 안경이나 하드렌즈를 착용했거나, 동공이 작거나, 시선이 불안정하거나, 표정이 풍부하다면 예외적 결과가 도출될 수 있음

========================================

[주의 사항 및 참고 사항]

아이트래킹은 비용과 숙련된 전문가가 아니라면, 유의미한 데이터 수집에 어려움이 존재함, 따라서 꼭 필요한 상황인지 고려하여 결정해야 함

기본적인 사용성 테스트를 충분히 실시한 후 추가 리서치가 필요한 세부사항이 있다면 그때 진행함

질적 사용성 테스트와 병행하면 한층 의미 있는 결과를 얻을 수 있음

참가자가 아이트래킹 기구를 착용하고 세션을 진행하는 동안 방해받지 않아야 정확한 데이터를 얻을 수 있음. 따라서 사용성 테스트에서 강조하는 Think aloud는 활용하지 않아야 함

========================================

[아이트래킹 진행 과정 가이드]

준비(1~3주)
- 리서치 계획서 작성 (리서치 질문, 테스트 형식)
- 사용성 평가지표 결정하기
- 참가자 조건과 샘플링 숫자 확정
- 참가자에게 아이트래킹 기법 설명하고 동의 구하기
- 리서치 랩에 아이트래킹 기기 설치
- 파일럿 테스트

테스트하기(2~4일)
- 사전 준비(비밀유지계약서 작성, 레코딩 기기 설치)
- 참가자에게 아이트래킹 기기 착용시킨 후 연습 태스크 실시하기
- 시나리오와 태스크 전달하고 진행 (세션: 15~90분)
- 설문조사 (선택사항)
- 관찰자들과 디브리핑

분석하기(2~3주)
- 데이터 분석
- 측정 기준에 맞추어 패턴 찾기
- 리포트 준비 및 발표

========================================

[사용성 평가지표 결정]

1. 히트맵 (Heat Map)
참가자의 시선이 어디를 향하는지, 얼마나 오래 머무는지 컬러를 이용해 시각적으로 보여줌

2. 게이즈 플롯 (Gaze Plot)
안구가 어떤 방향과 순서로 움직이는지 추적. 보통 참가자의 시선은 잠시 동안 주목했다가 빠르게 이동한다. 붉은 점이 집중한 부분이며 붉은 선은 시선의 이동 경로를 나타냄

========================================

[아이트래킹 데이터 분석 지표]

1. 시선 고정 (Fixation)
- 시선이 특정 위치에 머무르는 시간
- 일반적으로 100-300ms 이상 머무르는 경우를 시선 고정으로 정의
- 시선 고정 횟수와 지속 시간 분석

2. 시선 이동 (Saccade)
- 한 시선 고정에서 다음 시선 고정으로 이동하는 과정
- 시선 이동 거리와 속도 분석
- 시선 이동 패턴 파악

3. 시선 경로 (Scan Path)
- 사용자가 화면을 탐색하는 전체적인 시선 이동 경로
- 탐색 효율성과 전략 분석

4. 관심 영역 (Area of Interest, AOI)
- 분석하고자 하는 특정 화면 영역
- 각 AOI별 시선 고정 시간과 횟수 분석

========================================

[아이트래킹 실험 설계 고려사항]

1. 참가자 선정
- 시력이 정상인 참가자 선정
- 안경이나 콘택트렌즈 착용자도 가능하지만 결과 해석 시 고려 필요
- 충분한 수의 참가자 확보 (최소 20-30명)

2. 실험 환경
- 조명 조건 통제
- 화면 크기와 해상도 일정하게 유지
- 외부 방해 요소 최소화

3. 태스크 설계
- 자연스러운 사용 시나리오 기반
- 명확하고 구체적인 목표 제시
- 시간 제한 고려

4. 데이터 품질 관리
- 캘리브레이션 정확도 확인
- 데이터 손실 최소화
- 이상치 식별 및 제거

========================================

[아이트래킹 결과 해석 가이드]

1. 히트맵 해석
- 핫스팟(빨간색 영역): 높은 시선 집중도
- 콜드스팟(파란색 영역): 낮은 시선 집중도
- 시각적 계층구조와 정보 우선순위 파악

2. 게이즈 플롯 해석
- 시선 이동 패턴 분석
- 탐색 효율성 평가
- 사용자 행동 전략 이해

3. 통계적 분석
- 그룹 간 비교 분석
- 통계적 유의성 검증
- 효과 크기 측정

========================================

[아이트래킹과 다른 방법론과의 결합]

1. 사용성 테스트와 결합
- 정량적 시선 데이터 + 정성적 사용자 피드백
- 더 풍부한 인사이트 도출

2. 설문조사와 결합
- 시선 행동과 주관적 만족도 상관관계 분석
- 사용자 선호도와 실제 행동 비교

3. 인터뷰와 결합
- 시선 패턴에 대한 사용자 설명 수집
- 행동의 의도와 동기 파악

========================================

[아이트래킹 활용 사례]

1. 웹사이트 디자인 최적화
- 메뉴 구조와 네비게이션 개선
- 콘텐츠 배치 최적화
- CTA 버튼 위치 조정

2. 광고 효과 측정
- 광고 주목도 분석
- 브랜드 인지도 측정
- 광고 메시지 전달력 평가

3. 제품 패키징 디자인
- 제품 정보 가독성 개선
- 브랜드 요소 강화
- 구매 의도 유도 요소 분석

4. 모바일 앱 UX 개선
- 터치 영역 최적화
- 정보 계층구조 개선
- 사용자 플로우 최적화
